# 📡 SaaradhiGO Driver API Documentation

This document provides details for the backend API endpoints used by the SaaradhiGO Driver application.

## 🚀 Base URL
- **Local:** `http://localhost:8000/api/v1`
- **Production:** `https://api.saaradhigo.com/api/v1`

---

## 🔐 Authentication
All non-auth requests require a Bearer JWT Token in the header:
`Authorization: Bearer <your_token>`

### 1. Request OTP
- **Endpoint:** `POST /auth/otp`
- **Body:** `{ "phone": "91xxxxxxxxxx" }`
- **Description:** Sends a 6-digit OTP to the driver's phone.

### 2. Verify OTP / Login
- **Endpoint:** `POST /auth/login`
- **Body:** `{ "phone": "91xxxxxxxxxx", "otp": "123456" }`
- **Response:** `{ "token": "JWT_TOKEN", "driver": { ... } }`

---

## 🚗 Driver Management

### 1. Get Profile
- **Endpoint:** `GET /driver/profile`
- **Description:** Returns the complete driver profile, rating, and status.

### 2. Update Status (Online/Offline)
- **Endpoint:** `PATCH /driver/status`
- **Body:** `{ "isOnline": true }`
- **Description:** Toggles the driver's availability for rides.

### 3. Upload Document
- **Endpoint:** `POST /driver/documents`
- **Body:** `Multipart FormData` (type, file)
- **Description:** Uploads Aadhaar, License, or RC for admin approval.

---

## 🗺️ Ride Management

### 1. Get Active Ride
- **Endpoint:** `GET /rides/active`
- **Description:** Returns the current ride if the driver is in a trip.

### 2. Update Ride Status
- **Endpoint:** `PATCH /rides/:rideId/status`
- **Body:** `{ "status": "ARRIVED" | "IN_PROGRESS" | "COMPLETED" }`
- **Description:** Advances the trip lifecycle.

### 3. Heatmap Data
- **Endpoint:** `GET /rides/heatmap`
- **Description:** Returns high-demand clusters (Lat/Lng with intensity) for the map.

---

## 💰 Payments & Wallet

### 1. Get Wallet Balance
- **Endpoint:** `GET /wallet/balance`
- **Response:** `{ "balance": 1250.50 }`

### 2. Transaction History
- **Endpoint:** `GET /wallet/transactions`
- **Description:** List of credits (earnings) and debits (payouts).

---

## ⚠️ Safety (SOS)
- **Endpoint:** `POST /rides/:rideId/sos`
- **Body:** `{ "lat": 17.44, "lng": 78.39 }`
- **Description:** Triggers emergency alerts to admin and contacts.

---

## 📡 WebSockets (Socket.IO)
- **Events:**
  - `register_driver`: Joins the driver to their private room.
  - `update_location`: Periodic GPS updates from the app.
  - `new_ride_request`: Sent from server when a ride is assigned.
  - `trip_update`: Status sync between Driver and Rider.
